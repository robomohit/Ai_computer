## 2026-04-21 - [Clean Stream UI]
**Learning:** Text-based icons (`=` and `x`) paired with missing `aria-label`s hurt both aesthetic consistency and screen reader usability. Bulky box-shadows and thick borders on chat/stream UI components often contribute to cognitive overload.
**Action:** Always replace purely structural text pseudo-icons with semantic SVGs that gracefully handle color-inversion and hover states. When updating streaming UIs (like chat), default to flatter layouts (removing heavy borders and drop shadows) to emulate modern clean patterns (e.g. Claude Code) while maintaining distinct backgrounds to separate context zones.
## 2025-05-30 - Added ARIA Labels to Standalone Inputs
**Learning:** Native standalone inputs without explicit text labels (e.g., search bars, command palette inputs) rely solely on placeholders, which screen readers may skip or misinterpret. Playwright's `page.accessibility.snapshot()` is not available in our headless test sandbox, necessitating the use of `locator.get_attribute("aria-label")` for programmatic verification of accessibility improvements.
**Action:** When adding ARIA labels in vanilla HTML contexts, always write a targeted Playwright assertion using `get_attribute` instead of relying on broader accessibility snapshot features, and ensure inline `div`-based labels are converted to semantic `<label>` tags while using CSS (`display: block;`) to preserve their intended layout.
